**Use case _«Восстановление пароля зарегистрированного пользователя по номеру телефона»_**

| №           | Name1   | Содержание   |
| ----------- | ----------- | ----------- |
| 1           | Действующие лица  |Пользователь, система, электронная почта, телефон |
| 2           | Цель  |Установить новый пароль для зарегистрированного пользователя с помощью номера телефона |
| 3           | Запускающее событие  |Пользователь нажал кнопку "Забыл пароль" в окне авторизации |
| 4           | Основной сценарий  | Пользователь нажал кнопку "Забыл пароль" в окне авторизации; <br> Система открывает форму ввода номера телефона/электронной почты; <br> Система определяет введен адрес электронной почты или номер телефона; <br> Пользователь вводит номер телефона и нажимает "Ок"; <br> Система выводит сообщение об отправке сообщения с кодом на указанный номер телефона на 7 секунд; <br> Система отправляет сообщение с кодом на указанный номер телефона; <br> Система открывает пользователю окно подтверждения номера телефона с полем для ввода направленного кода; <br>  Пользователь вводит код из сообщения и нажимает «Ок»; <br> Система проверяет корректность введенного кода; <br> Система открывает окно смены пароля; <br> Пользователь вводит новый пароль и нажимает «Ок»; <br> Система выводит сообщение об успешной смене пароля на 7 секунд; <br> Система открывает окно авторизации.  |
| 5           | Исключения  |1.	Введен не верный код для подтверждения номера телефона. Система выдает сообщение о некорректности введенного кода. <br> 2.	Пользователь вводит адрес электронной почты, который не зарегистрирован в системе. В таком случае система выдает сообщение, что указанный адрес электронной почты не зарегистрирован в системе. |
| 6           | Постусловия  |Пара «логин-пароль» в БД обновлены |
